The Pipeline

Nine gates between an idea and production.

Each gate is a real artifact, not a checkbox. Two gates carry CRITICAL severity — when they fail, nothing downstream runs.

01
Discovery Constitution
CRITICAL

Audience, jobs-to-be-done, four-risk inventory, and kill criteria written down before a single line ships. If the problem isn't real, the pipeline stops here.

02
Statement of Work
Scope

Shape Up–style appetite, fixed boundaries, explicit non-goals. The scope is what gets cut, not what gets added.

03
Design
Artifact

PR-FAQ, flows, and the smallest possible surface that proves the bet. No screens you can't justify against the JTBD.

04
Build
Implementation

Vertical slice with tests-first discipline and architecture decisions captured as ADRs. The code is the easy part now.

05
QA
CRITICAL

Adversarial review across correctness, security, performance, and accessibility. CRITICAL findings block deploy. No exceptions.

06
UAT
Validation

Real users running the real flow against the success criteria written at Gate 1. Demo theatre doesn't count.

07
Deploy
Release

Reproducible build, rollback path, observability wired in before the URL goes public. Boring on purpose.

08
Handoff
Operations

Runbooks, dashboards, on-call surface area documented. The team that owns it tomorrow is set up to win.

09
Warranty
Post-launch

A defined window where the original team owns regressions and outcome metrics. The launch isn't the finish line — the metric is.
