The pipeline

9 gates. Every one blocks.

Each gate is a Python audit script with real pass/fail criteria. Gates 1 and 5 are CRITICAL — one CRITICAL finding halts everything.

Gate 01
Discovery Constitution
JTBD + four-risk model. Who gets fired if this fails? What does failure look like in production?
● CRITICAL
Gate 02
Statement of Work
Shape Up appetite + PR-FAQ. Scope is locked. Deferred list is explicit.
Blocking
Gate 03
Design
Every screen maps to a job defined in Gate 1. No orphan screens.
Blocking
Gate 04
Build
Commit-to-AC linkage required. No orphan TODOs allowed in shipped code.
Blocking
Gate 05
QA
Unit + integration + E2E — all must be green. One red test = full stop.
● CRITICAL
Gate 06
UAT
A real human must sign off on a real URL. No synthetic sign-offs.
Blocking
Gate 07
Deploy
HTTP 200 to prod, smoke job passes, rollback drill completed.
Pass/Fail
Gate 08
Handoff
ADRs + runbook + DEBT.md. A contract, not a README.
Blocking
Gate 09
Warranty
72-hour monitoring window. Error rate, latency, uptime — all tracked.
Pass/Fail
